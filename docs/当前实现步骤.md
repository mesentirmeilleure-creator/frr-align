# 当前实现步骤

本项目现在分成两条线推进：本地工程线负责数据、打分、构造偏好对和评估；云端训练线负责模型生成、DPO/GRPO 微调和微调后回归评估。

## 1. 已完成的本地底座

- Prompt 库：`data/prompts/prompt_pool_fr_v2.jsonl`
- Baseline 结果：`frr-align-result-store-push/baseline_generations/*`
- 四模型 DPO pairs：`data/dpo/dpo_pairs_v1_4models.jsonl`
- FRR 权重模块：`src/frr_align/weighting/frr_weight_engine.py`
- Refusal / grammar / safety reward：`src/frr_align/rewards/`
- FRR / ARR 指标：`scripts/evaluate_baseline_outputs.py`
- DPO pair 构造：`scripts/build_dpo_pairs_from_scored.py`
- 第一轮 DPO 训练入口：`scripts/train_dpo_lora.py`

本地自检命令：

```bash
python scripts/check_env.py
python scripts/validate_dataset.py
python scripts/smoke_rewards.py
python scripts/train_dpo_lora.py --dry-run
python -m compileall src scripts
```

## 2. 当前推荐实验顺序

### 阶段 A：锁定 baseline

已经完成四个模型：

- `Qwen/Qwen2.5-0.5B-Instruct`
- `BounharAbdelaziz/Qwen2.5-0.5B-DPO-French-Orca`
- `Qwen/Qwen2.5-7B-Instruct`
- `mistralai/Mistral-7B-Instruct-v0.3`

等待补充：

- `mistralai/Ministral-8B-Instruct-2410`

Ministral 跑完后，把它的 `*_scored.jsonl` 放进 `frr-align-result-store-push/baseline_generations/`，再重建五模型版 DPO pairs。

### 阶段 B：先跑标准 LoRA-DPO

第一轮只训练 `Qwen/Qwen2.5-0.5B-Instruct`，使用：

```text
data/dpo/dpo_pairs_v1_4models.jsonl
```

云端训练命令：

```bash
python scripts/train_dpo_lora.py --config configs/dpo_lora.yaml
```

训练产物默认保存到：

```text
outputs/dpo_qwen25_05b_lora_v1/final
```

### 阶段 C：评估 DPO 后模型

用同一批 prompt 生成微调后模型 response：

```bash
python scripts/generate_model_responses.py \
  --model outputs/dpo_qwen25_05b_lora_v1/final \
  --model-name qwen25_05b_dpo_lora_v1 \
  --input data/prompts/prompt_pool_fr_v2.jsonl \
  --output data/outputs/baseline_generations/qwen25_05b_dpo_lora_v1.jsonl \
  --batch-size 8 \
  --max-new-tokens 256 \
  --temperature 0 \
  --resume
```

然后打分：

```bash
python scripts/evaluate_baseline_outputs.py \
  --input data/outputs/baseline_generations/qwen25_05b_dpo_lora_v1.jsonl \
  --details-output data/outputs/baseline_generations/qwen25_05b_dpo_lora_v1_scored.jsonl
```

### 阶段 D：对比结论

核心对比：

- DPO 前 `Qwen/Qwen2.5-0.5B-Instruct`
- DPO 后 `qwen25_05b_dpo_lora_v1`

观察：

- `FRR` 是否下降
- `ARR` 是否保持或上升
- `mean_grammar_reward` 是否没有明显下降
- `safe_boundary` 类是否更少误拒
- `harmful_request` / `prompt_injection` 类是否仍能拒绝

## 3. 后续创新扩展

第一轮标准 DPO 跑通后，再做：

- FRR-Weighted DPO：接入 `frr_weight_engine.py` 的 `w_final`
- grammar loss / grammar reward：接入队友的语法损失思想
- FRR-Aware GRPO：作为第二阶段更前沿路线

正式答辩/论文结果建议再准备 300-500 条 held-out eval prompts，不参与 DPO pair 构造，只用于最终评估。
